# Live Portal Notes — learn.soic.in (Learnyst "Bodhi")

Captured from a logged-in session on 2026-06-20 via an interactive browser. These
are the *real* facts about the portal, replacing the best-effort guesses the
original `crawler.py`/`extract.py` were built on.

## Platform

- **Stack:** Next.js front-end + Learnyst **"Bodhi" Web Components** (custom
  elements named `BODHI-*`). PWA with an active **service worker** (offline
  caching) and content protection.
- **School id:** `110998` (seen on `events.learnyst.com`).
- **Asset/version:** Bodhi widgets served from `edge-cdn.learnyst.com/...bodhi/prod/next-js/vX.Y.Z/`.

## Why the original approach cannot work

| Original assumption | Reality |
|---|---|
| Pages expose `<a href>` to courses/lessons | **Zero content anchors.** Cards are `BODHI-STORE-CARD` / `BODHI-BUNDLE-CONTENT` elements navigated by JS click handlers (framework routing). Only nav/footer are real `<a>`. |
| `extract_lesson(page.content())` parses HTML | All content lives in **Shadow DOM**. `page.content()` does **not** serialize shadow roots → extraction gets nothing. Must pierce shadow DOM (recurse `el.shadowRoot`). Playwright CSS selectors pierce open shadow DOM; raw HTML parsing (BeautifulSoup on `page.content()`) does not. |
| Hierarchy is Course → Module → Lesson | Real hierarchy is **Bundle → Course → Section → Lesson** (sections are the natural "modules"). |
| Lesson body is static text | Lessons are typed **Video / Article**; bodies and transcripts load asynchronously and are **protected** (see below). |

## Hierarchy & URL patterns

- Dashboard: `https://learn.soic.in/learn` — enrolled items in
  `BODHI-STORE-ENROLLED-LIST` (e.g. the **SOIC Membership** bundle, "14 Lessons").
- Bundle page: `https://learn.soic.in/learn/home/{bundleSlug}`
  (the membership bundle slug is `SOIC-Course`; sidebar: "15 Courses").
  Course cards live in `BODHI-BUNDLE-CONTENT`; each shows `Course • N lessons`.
- Lesson page (canonical):
  `https://learn.soic.in/learn/home/{bundleSlug}/{courseSlug}/section/{sectionId}/lesson/{lessonId}`
  - Visiting `/section/0/lesson/0` redirects to the first real numeric
    `{sectionId}/{lessonId}` (e.g. `/section/589476/lesson/4830058`).
  - **Section & lesson IDs are NOT present in the DOM** (no `data-*`/`href`).
    They only appear in the URL after the framework navigates. So enumerating
    lessons requires **click-driven navigation** (click a syllabus row → read
    `page.url`) or the private Learnyst API (SW-opaque; not used here).

## Lesson page anatomy (right sidebar tabs)

- **Syllabus** — sections + lessons. Rendered text per section header parses with:
  `^(\d+)\s*(.+?)\s*(\d+)\s*lessons?(?:\s*[•·]\s*(\d+)\s*attachments?)?$`
  Lesson rows render as `"<title><type> • <duration|attachment>"`, type ∈
  {`video`, `article`}, e.g. `Video • 4m 30s`, `Article • attachment`.
- **Transcript** (`BODHI-LESSON-TRANSCRIPT`) — verbatim video transcript; loads
  only on playback. In open shadow DOM (not an iframe).
- **Summary** — **AI-generated** lesson summary ("Lesson summary generated by
  AI"); loads into open shadow DOM, ~few KB of clean prose, no iframe.

## Content protection (decisive for scope)

- A persistent banner renders: **"Downloading or sharing content is prohibited."**
- Lesson bodies are watermarked with the viewer's identity (email + phone).
- **Article bodies render inside a sandboxed `srcdoc` iframe**: parent JS and the
  accessibility tree **cannot** read it (`get_page_text` returns everything *except*
  the body). Extracting it requires defeating the sandbox = **circumvention**.

### Capture-scope decision (personal study; honor guardrails)

| Content | Reachable without circumvention? | Decision |
|---|---|---|
| Structure (bundle/course/section/lesson, title, type, duration, attachment flag) | Yes (open DOM) | **Capture** |
| AI Summary (per lesson) | Yes (open shadow DOM, derived/transformative) | **Capture** |
| Article bodies | No (sandboxed iframe + watermark + prohibition) | **Do not capture** |
| Verbatim transcript | Technically yes, but it is the protected video's content in text form | **Do not capture** (no ripping of protected A/V) |

## Extraction recipe (proven in-session)

```js
// Recurse open shadow roots — Playwright: run via page.evaluate
function* walk(root){ for(const el of root.querySelectorAll('*')){ yield el;
  if(el.shadowRoot) yield* walk(el.shadowRoot); } }
```
- Sections: scan `walk(document)` text with the section regex above.
- Lesson rows: elements matching `/(Video|Article)\s*[•·]/` with few children.
- AI summary: open the **Summary** tab, then take the largest non-`<style>`/`<script>`
  text block under the summary host.
- Lesson URL: click the syllabus row, read `location.href`.
```
